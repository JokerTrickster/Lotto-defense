# Lotto Defense - 게임 디자인 문서 (GDD)

## 1. 게임 컨셉

**Lotto Defense**는 랜덤 유닛 소환(가챠)과 조합 시스템을 결합한 모바일 타워디펜스 게임이다. 플레이어는 골드를 사용해 랜덤 유닛을 소환하고, 4x5 그리드에 배치하여 웨이브로 몰려오는 몬스터를 막는다. 같은 유닛 2개를 조합하면 상위 유닛으로 진화한다.

### 핵심 루프

```
소환 (가챠) → 배치 (그리드) → 전투 (자동) → 조합 (진화) → 강화 (업그레이드) → 반복
```

### 게임 모드
- **싱글플레이**: 10라운드 생존
- **Co-op (2인)**: WebSocket 기반 실시간 병렬 플레이, 각자의 맵에서 독립 진행, 상태 공유

## 2. 유닛 시스템

### 2.1 유닛 종류 (5종)

| 유닛 | 등급 | 공격력 | 공속 | 사거리 | 공격 패턴 | 스킬 |
|------|------|--------|------|--------|-----------|------|
| 전사 (Warrior) | Normal | 10 | 1.0 | 1.5 | SingleTarget | Critical Strike |
| 궁수 (Archer) | Rare | 15 | 1.2 | 3.0 | Pierce | Multi Shot |
| 마법사 (Mage) | Epic | 25 | 0.8 | 4.0 | AOE | Chain Lightning |
| 용기사 (DragonKnight) | Legendary | 40 | 0.7 | 2.0 | Splash | War Cry |
| 피닉스 (Phoenix) | Legendary | 50 | 0.6 | 5.0 | Chain | Meteor |

### 2.2 공격 패턴

| 패턴 | 설명 |
|------|------|
| SingleTarget | 단일 대상에 전체 데미지 |
| Pierce | 15도 원뿔 범위, 최대 N개 대상 관통 |
| Splash | 주 대상 전체 데미지 + 주변 감소 데미지 (거리 비례 감소) |
| AOE | Splash와 동일한 범위 감소 |
| Chain | 연쇄 공격, 튕길 때마다 데미지 20% 감소 |

### 2.3 스킬 시스템

**스킬 타입:**

| 타입 | 발동 조건 |
|------|-----------|
| Active | 마나 100 충전 시 자동 발동 (쿨다운 기반 마나 회복) |
| Passive | 상시 적용 |
| OnHit | 공격 적중 시 확률/쿨다운 발동 |
| OnKill | 적 처치 시 발동 |

**유닛별 스킬:**

| 유닛 | 스킬 | 타입 | 효과 |
|------|------|------|------|
| 전사 | Critical Strike | OnHit | 쿨다운 3초, 데미지 2배 |
| 궁수 | Multi Shot | Active | 쿨다운 12초, 공속 2배 + 슬로우 0.5x (3초) |
| 마법사 | Chain Lightning | OnKill | 쿨다운 2초, 3개 대상 연쇄 (0.5x 데미지) |
| 용기사 | War Cry | Active | 쿨다운 10초, 데미지 2배 (3초 지속) |
| 피닉스 | Meteor | Active | 쿨다운 15초, 데미지 3배 + 프리즈 2초 |

### 2.4 소환 (가챠)

- 비용: **5 골드**
- 확률: Normal 25%, Rare 25%, Epic 25%, Legendary 25%
- 해금된 유닛만 소환 풀에 포함
- 초기 해금: Warrior만 (상점에서 추가 해금)

### 2.5 조합 체인

같은 유닛 2개를 조합하면 상위 유닛 1개로 진화:

```
전사 (Normal) → 궁수 (Rare) → 마법사 (Epic) → 용기사 (Legendary) → 피닉스 (Legendary)
```

- 조합 비용: 0 골드
- 업그레이드 레벨 중 높은 것 계승
- 자동 조합 기능 지원

### 2.6 업그레이드

- 공격력/공속 별도 업그레이드
- **등급별 레벨 공유** (같은 등급 유닛은 업그레이드 공유)
- 최대 레벨: 10
- 공격력: 레벨당 +10% (Phoenix: +12%)
- 공속: 레벨당 +8% (Phoenix: +10%)
- 비용: `기본비용 × (1 + 현재레벨 × 0.5)`

## 3. 몬스터 시스템

### 3.1 몬스터 종류

| 타입 | HP | 방어 | 속도 | 골드 | HP 스케일링 | 방어 스케일링 |
|------|-----|------|------|------|-------------|--------------|
| Normal (고블린) | 100 | 5 | 2.0 | 10 | ×1.1/라운드 | ×1.05/라운드 |
| Fast (스피드데몬) | 70 | 3 | 4.0 | 8 | ×1.08/라운드 | ×1.03/라운드 |
| Tank (아머드오거) | 200 | 10 | 1.5 | 15 | ×1.12/라운드 | ×1.06/라운드 |
| Boss (드래곤보스) | 기본×10 | 기본×3 | 기본×0.7 | 기본×20 | - | - |

### 3.2 보스 규칙
- 5라운드에 등장
- 보스 스탯: HP 10배, 방어 3배, 속도 0.7배, 공격 5배, 골드 20배
- 등장 시 2초 딜레이 후 스폰

### 3.3 이동 경로
- 4x5 그리드 외곽을 사각형 루프로 이동
- 중간 웨이포인트 포함
- 경로 끝 도달 시 플레이어 생명 감소

### 3.4 CC (군중 제어)
- **슬로우**: 이동속도 감소 (배수 적용), 중복 시 더 강한 값 적용
- **프리즈**: 이동 완전 정지, 중복 시 더 긴 시간 적용

## 4. 전투 시스템

### 4.1 전투 흐름
- Tick 기반: 0.1초 간격 (초당 10회)
- 자동 타겟팅: 사거리 내 가장 가까운 몬스터
- 사거리 이탈 시 20틱 후 타겟 해제

### 4.2 데미지 공식
```
실제 데미지 = max(1, 유닛 공격력 - 몬스터 방어력)
```

### 4.3 유닛 공격력 계산
```
최종 공격력 = 기본 공격력 × 업그레이드 배율 × 로비 레벨 배율 × 액티브 스킬 버프
```

---

## 5. 라운드 시스템

### 5.1 라운드 흐름

```
카운트다운 (3초) → 준비 (15초) → 전투 (30초) → 결과 → 다음 라운드
```

- 총 10라운드 (설정상 30라운드까지 확장 가능)
- 준비 시간: 15초 (유닛 소환/배치/조합/업그레이드)
- 전투 시간: 30초 (자동 전투)

### 5.2 몬스터 스폰
- 라운드당 기본 30마리
- 스폰 간격: 0.5초
- 스폰 지속: 15초
- 최대 동시 활성: 100마리

### 5.3 난이도

| 난이도 | HP 배율 | 방어 배율 | 골드 배율 |
|--------|---------|----------|----------|
| 보통 (Normal) | 1.0x | 1.0x | 1.0x |
| 어려움 (Hard) | 1.5x | 1.3x | 1.5x |
| 매우 어려움 (VeryHard) | 2.0x | 1.5x | 2.0x |

추가로 DifficultyConfig의 AnimationCurve가 라운드별 HP(1→5배)/방어(1→3배) 스케일링 적용.

## 6. 그리드 시스템

- 크기: 4열 × 5행 = 20칸
- 모든 칸에 유닛 배치 가능
- 유닛 크기: 셀 크기 × 0.8
- 셀 상태: Normal, Hover, Selected, Invalid, Occupied
- 유닛 이동/교체 지원

---

## 7. 경제 시스템

### 7.1 골드 수입
- 시작 골드: 30
- 몬스터 처치 골드: 몬스터별 상이 (8~15, 보스 ×20)
- 게임 결과 보상: 라운드 도달에 따라 10~100골드

### 7.2 골드 지출
- 소환: 5골드
- 업그레이드: 등급별 기본비용 × (1 + 레벨 × 0.5)
- 조합: 무료

### 7.3 유닛 판매 가격
| 등급 | 판매 가격 |
|------|----------|
| Normal | 3골드 |
| Rare | 8골드 |
| Epic | 20골드 |
| Legendary | 50골드 |

## 8. 메타 시스템

### 8.1 상점
유닛 해금 (영구):

| 유닛 | 가격 |
|------|------|
| Warrior | 무료 |
| Archer | 100골드 |
| Mage | 500골드 |
| DragonKnight | 2,000골드 |
| Phoenix | 3,000골드 |

### 8.2 일일 보상
| 클리어 횟수 | 보상 |
|------------|------|
| 1회 | 50골드 |
| 3회 | 티켓 1장 |
| 5회 | 150골드 |
| 10회 | 200골드 + 티켓 2장 |

### 8.3 퀘스트
- **CollectUnits**: 특정 유닛 N마리 배치 (예: 전사 3마리)
- **PositionUnits**: 특정 위치에 유닛 배치 (예: 마법사를 (0,0)에)
- 상태: Hidden → Completed → Rewarded
- 보상: 골드 + 프로필 아바타 해금

### 8.4 프로필
- 닉네임 (2~12자)
- 아바타 7종: 기본 3종 (기본, 붉은전사, 숲의수호자) + 퀘스트 해금 4종
- 등급: Common, Uncommon, Rare, Epic, Legendary, Hidden

### 8.5 우편함
- 공지 메일, 보상 메일 수신

## 9. 멀티플레이어

### 9.1 구조
- 2인 병렬 개인전
- WebSocket 실시간 연결 + REST 폴링 (3초)
- 각자의 맵에서 독립 진행

### 9.2 흐름
1. 방 생성/참가/자동 매칭
2. 양쪽 준비 완료
3. 게임 시작 (동시)
4. 상태 동기화 (생명, 라운드, 골드, 처치수, 유닛수)
5. 한쪽 탈락 또는 10라운드 종료 시 결과 판정

### 9.3 승리 조건
- 상대방이 먼저 탈락하면 승리
- 둘 다 생존 시 라운드, 처치 수 등으로 판정

- x1 (기본), x2, x3 배속
